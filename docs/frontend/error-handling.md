# 프론트 에러 처리 기준

## 공통 에러 포맷
백엔드는 다음 형식으로 에러를 반환합니다.

```json
{
  "success": false,
  "code": "AUTH_001",
  "message": "이메일 또는 비밀번호가 올바르지 않습니다.",
  "data": null
}
```

## 화면 처리 원칙
- 입력값 오류: 해당 input 아래에 메시지를 보여줍니다.
- 인증 만료: 재발급 시도 후 실패하면 로그인 화면으로 이동합니다.
- 권한 없음: 역할에 맞는 로그인 화면 또는 홈으로 이동합니다.
- Rate Limit: `Retry-After` 헤더가 있으면 잠시 후 재시도 안내를 보여줍니다.
- 서버 장애: 토스트 또는 공통 에러 박스를 보여줍니다.

## 프론트 구현 권장
- API client에서 `success === false`를 공통 예외로 변환합니다.
- 화면 컴포넌트는 `error.message`만 표시하도록 단순화합니다.
- 에러 코드별 분기는 `error.code` 기준으로 중앙화합니다.
